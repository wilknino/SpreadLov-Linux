## 🔥 FIREBASE CONFIGURATION (Server Side)

### **STEP 6: Add Firebase Credentials to Replit**

Your Replit server needs Firebase credentials to send push notifications.

1. **Get Firebase Credentials:**
   - Go to Firebase Console: https://console.firebase.google.com
   - Select your project
   - Click the gear icon ⚙️ → Project Settings → Service Accounts
   - Click "Generate New Private Key"
   - Download the JSON file

2. **Extract values from the JSON file:**

Open the downloaded JSON file. You'll see something like:

```json
{
  "type": "service_account",
  "project_id": "spreadlov-12345",
  "private_key_id": "abc123...",
  "private_key": "-----BEGIN PRIVATE KEY-----\nMIIEvQIB...\n-----END PRIVATE KEY-----\n",
  "client_email": "firebase-adminsdk-xyz@spreadlov-12345.iam.gserviceaccount.com",
  ...
}
```

You need these 3 values:
- `project_id` → Copy this
- `client_email` → Copy this
- `private_key` → Copy this (including the `-----BEGIN` and `-----END` lines)

3. **Add to Replit Secrets:**
   - In Replit, click the "Secrets" tab (🔒 icon on left sidebar)
   - Add 3 new secrets:

| Secret Key | Value |
|------------|-------|
| `FIREBASE_PROJECT_ID` | Your `project_id` from JSON |
| `FIREBASE_CLIENT_EMAIL` | Your `client_email` from JSON |
| `FIREBASE_PRIVATE_KEY` | Your `private_key` from JSON (keep the \n characters) |

4. **Restart Replit Server:**
   - Click "Stop" in Replit console
   - Click "Run" again
   - You should see: **"✅ Firebase Admin initialized successfully"**

## 📊 HOW IT WORKS (Technical Overview)

### **Complete Flow for Each Notification Type:**

#### **Profile View Notification:**
1. User B views User A's profile on website
2. Server API creates notification in database (type: `profile_view`)
3. PostgreSQL trigger fires → sends event to notification listener
4. Notification listener fetches User A's FCM token
5. FCM service sends push to User A's Android phone
6. Notification appears: "User B viewed your profile"
7. User A taps notification → Opens app → Goes to User B's profile

#### **Profile Like Notification:**
1. User B likes User A's profile on website
2. Server API creates notification in database (type: `profile_like`)
3. PostgreSQL trigger fires → sends event to notification listener
4. Notification listener fetches User A's FCM token
5. FCM service sends push to User A's Android phone
6. Notification appears: "User B liked your profile"
7. User A taps notification → Opens app → Goes to User B's profile

#### **Message Notification:**
1. User B sends message to User A
2. Server checks: Is User A actively chatting with User B?
3. If NO (app in background or on different chat):
   - Server creates notification in database (type: `message_received`)
   - PostgreSQL trigger fires → sends event to notification listener
   - Notification listener fetches User A's FCM token
   - FCM service sends push to User A's Android phone
   - Notification appears: "User B sent you a message"
4. If YES (User A is actively chatting):
   - NO notification created (user is already seeing messages)
5. User A taps notification → Opens app → Goes to chat with User B
